# Ruslan Nanava — лендинг-портфолио

Семантичная вёрстка лендинга по макету Figma + полная мобильная адаптация.
Чистый HTML + CSS + немного ванильного JS, без сборщиков и зависимостей.

## Структура проекта

```
ruslan-nanava-site/
├── index.html          # разметка страницы
├── css/
│   └── style.css       # стили + адаптив (дизайн-токены, медиа-запросы)
├── js/
│   └── main.js         # меню, плавная прокрутка, scroll-анимации
├── images/             # изображения
│   ├── hero.png            # фон геро-секции  ← ЗАМЕНИТЬ экспортом из Figma
│   ├── project-1.png       # проект 01 (планшет)        ← ЗАМЕНИТЬ
│   ├── project-2-soul.png  # проект 02 (ноутбук)        ← ЗАМЕНИТЬ
│   ├── project-3-poster.png# проект 03 (постеры)        ← ЗАМЕНИТЬ
│   ├── project-laptop.png  # проект 04 (ноутбук/кресло) ← ЗАМЕНИТЬ
│   └── grain.png           # текстура зернистости (готова, заменять не нужно)
└── README.md
```

## Как открыть

Просто откройте `index.html` в браузере. Сборка не требуется.
Для корректной работы относительных путей удобно поднять локальный сервер:

```bash
# из папки проекта
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## ⚠️ Важно: замена изображений

Бинарные ассеты дизайна (фотография в геро-секции и 4 мокапа проектов)
**нельзя выгрузить из Figma автоматически** — их нужно экспортировать вручную.
Сейчас на их месте стоят плейсхолдеры с правильными размерами и именами,
поэтому макет уже собирается и выглядит верно. Чтобы подставить реальную графику:

1. Откройте макет в Figma.
2. Выделите нужный слой/фрейм → панель **Export** (справа внизу) → формат **PNG** (или **WebP**), масштаб **2x** для чёткости на retina.
3. Сохраните файл **с тем же именем**, что и плейсхолдер, и положите в папку `images/`,
   заменив существующий. Имена менять не нужно — пути в HTML уже прописаны:

   | Слой в Figma                         | Имя файла                  |
   |--------------------------------------|----------------------------|
   | Фото в геро-секции                   | `hero.png`                 |
   | Проект 01 (планшет)                  | `project-1.png`            |
   | Проект 02 (ноутбук Soul)             | `project-2-soul.png`       |
   | Проект 03 (постеры)                  | `project-3-poster.png`     |
   | Проект 04 (ноутбук в кресле)         | `project-laptop.png`       |

> Текстуру `grain.png`, рамку блока услуг и разделительные линии воссозданы
> средствами CSS/SVG — экспортировать их из Figma не нужно.

## Адаптивность

Вёрстка резиновая (типографика на `clamp()`), плюс контрольные точки:

| Ширина        | Поведение                                                        |
|---------------|------------------------------------------------------------------|
| > 900px       | десктоп: сетка проектов 2×2, услуги 2×2 с крестом-разделителем    |
| ≤ 900px       | планшет: уменьшаются отступы и смещения                          |
| ≤ 900px       | планшет: десктоп-навигация прячется, появляется бургер |
| ≤ 768px       | мобайл: проекты в 1 колонку, услуги в 1 колонку, подвал в 2/1 колонки |
| ≤ 420px       | малые телефоны: уменьшенный трекинг, выравнивание текста по левому краю |

## Дизайн-токены (CSS-переменные в `:root`)

- Цвета: `--dark #3a3a3a`, `--gray #5e5e5e`, `--gray-light #9b9b9b`, `--card-bg #f3f3f3`
- Шрифт: **Manrope** (подключён с Google Fonts, веса 200–600)
- Поля страницы: `--pad` (адаптивный отступ)
- Макс. ширина контента: `--maxw: 1570px` (как в макете)

## Что использовано

- Семантические теги: `header`, `nav`, `main`, `section`, `article`, `footer`
- CSS Grid и Flexbox для раскладки
- `clamp()` для плавной адаптивной типографики
- `aspect-ratio` для пропорций геро-секции и карточек
- Доступность: `aria`-атрибуты на меню, поддержка `prefers-reduced-motion`, закрытие меню по `Esc`

## Header и footer (в духе студийных сайтов)

- **Header** фиксированный и минималистичный: компактный логотип слева, текстовые
  ссылки справа («Обсудить проект» — акцентная ссылка с точкой, как «Get in Touch»
  у студийных сайтов). Над тёмным героем шапка прозрачная с белым текстом, а как только
  герой уходит вверх — становится светлой с размытием и тонкой линией снизу
  (переключение через `IntersectionObserver`, класс `at-top` на `<html>`).
  Без JS шапка по умолчанию светлая и читаемая.
- **Мобильное меню** — полноэкранный оверлей (а не выпадающая панель): крупные ссылки
  с выезжающей анимацией, контакты внизу, блокировка прокрутки фона, закрытие по `Esc`,
  тапу по пункту и при возврате на десктоп. Это устранило баг, когда из свёрнутого
  меню выглядывал нижний пункт.
- **Footer** — аккуратная строка из 4 колонок (бренд + ИНН, навигация, контакты,
  ©год) с тонкой верхней линией; на узких экранах сворачивается в 2, затем в 1 колонку.

## Масштаб и отступы

- Вся композиция уменьшена примерно до **90%** через токен `--scale: 0.9`
  (умножает размеры шрифтов и элементов) и уменьшенную макс. ширину `--maxw: 1440px`.
  Чтобы вернуть исходный масштаб — поставьте `--scale: 1` и `--maxw: 1570px` в `:root`.
- Межблочные отступы заданы токеном `--section-gap: clamp(120px, 19vw, 320px)` —
  на ширине макета (~1570px) это даёт **≈300px**, как в Figma, и плавно ужимается
  на узких экранах. Значение легко поменять в одном месте (например на жёсткие `350px`).

## Анимации и прокрутка («премиальная» подача)

- **Плавная инерционная прокрутка** — библиотека [Lenis](https://github.com/darkroomengineering/lenis),
  подключается с CDN. Это *необязательная* зависимость: если скрипт не загрузится,
  включается нативная плавная прокрутка, сайт работает как обычно.
- **«Пропечатка» текста при пролистывании** — заголовки печатаются посимвольно
  (с мигающей кареткой), абзацы проявляются пословесно. Срабатывает при попадании
  блока в зону видимости через `IntersectionObserver`, каждый блок анимируется один раз.
- **Появление изображений** — мягкий фейд с лёгким зумом; линия в блоке «Обсудить проект»
  прочерчивается.
- **Корректная деградация:** контент изначально виден; скрытие под анимацию включается
  только если работает JS (класс `html.js`) и пользователь не включил «уменьшение движения».
  При `prefers-reduced-motion: reduce` все анимации отключаются, текст показывается сразу.

### Как настроить анимации
- Скорость «печати» — переменные `--stagger-char` / `--stagger-word` в `:root` (CSS)
  и константа `STAGGER_CHAR` в `js/main.js` (держите их согласованными).
- Класс на элементе задаёт тип анимации: `reveal-type` (посимвольно),
  `reveal-words` (пословесно), `reveal` (фейд блока), `reveal-media` (картинка),
  `reveal-line` (линия). Атрибут `data-caret` добавляет мигающий курсор заголовку.
